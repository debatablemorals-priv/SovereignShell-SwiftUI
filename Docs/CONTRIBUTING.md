# Contributing to SovereignShell

Thank you for contributing to SovereignShell.

## Development Workflow

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Ensure the project builds successfully
5. Submit a pull request

## Branch Rules

The `main` branch is protected.

Changes must be submitted through pull requests and must pass automated build checks before merging.

## Coding Standards

- Follow Swift style conventions
- Keep code deterministic
- Maintain compatibility with the AIS architecture
- Document major architectural changes

## Reporting Issues

If you discover a bug or security issue, please open an issue describing:

- What happened
- Expected behavior
- Steps to reproduce

Security issues should be reported privately when possible.
